# 正奇建设产品范围说明

## 1. 定位

TenderDoc-Generator 第一版是正奇建设专用投标文件生成系统，不做全行业通用标书软件。

系统目标是把正奇常投项目的招标文件解析、商务/技术/报价分卷生成、废标审查、DOCX/PDF 导出和历史资料复用做深做稳。

## 2. 第一版支持范围

### 市政工程

- 市政道路
- 排水、管网、道路附属设施
- 市政维修、养护、改造类项目

### 公路工程

- 公路改建
- 公路扩建
- 公路维修养护
- 路基、路面、桥涵附属及道路安全相关内容

### 交通安全设施

- 标志标线
- 护栏、防眩设施
- 隔离栅、轮廓标、道口标柱等附属设施
- 交通安全设施养护、维修、更换

## 3. 第一版不主动覆盖

以下项目类型暂不作为第一版主线能力：

- 房建总承包
- 园林绿化
- 水利水电
- 机电安装
- 政府采购服务类
- 纯货物采购
- 勘察设计、监理、咨询服务

这些方向可以保留扩展可能，但不能影响正奇市政/公路主线交付。

## 4. 文件生成策略

系统应先生成三个独立分卷：

- 商务文件
- 技术文件
- 报价文件

完整投标文件只是按需合并输出。因为不同电子投标平台可能要求：

- 商务、技术、报价三个文件分别上传
- 商务及技术合并，报价单独上传
- 三卷合并为一个 PDF 上传

因此 workflow、预览、下载和 DOCX/PDF 导出都应保持分卷优先。

## 5. 模板库策略

模板库优先沉淀正奇常用模板：

- 市政工程模板
- 公路改建/扩建模板
- 交通安全设施养护模板
- 商务资格文件模板
- 报价文件模板
- 正奇 DOCX 样式模板

模板 JSON 决定章节结构，prompt 不硬编码目录。招标文件决定响应要求，知识库提供内容素材和企业表达习惯。

## 6. 知识库策略

知识库优先录入和标注：

- 正奇已中标投标文件
- 公司资质、人员证书、业绩资料
- 市政道路施工方案
- 公路改扩建施工方案
- 交通安全设施养护方案
- 标志标线、护栏、防眩、隔离栅等专项方案
- 常见废标条款、评分办法和响应口径

知识库资料应按 `project_type`、`specialty`、`volume`、`region`、`year` 等标签管理。

## 7. 验收标准

第一版产品化验收不以“支持多少行业”为标准，而以正奇真实项目可用性为标准：

- 可识别市政/公路/交安/改扩建招标文件类型
- 可推荐对应模板
- 可独立生成商务、技术、报价三卷
- DOCX 格式接近正奇历史中标文件
- 审查报告能覆盖废标项、资质项、评分项
- 用户可人工编辑、确认、下载最终文件
